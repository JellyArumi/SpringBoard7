 프로젝트 개발 순서
 
 [Spring MVC]
 SQL(mapper) -> DAO/DAOImpl ->Service/ServiceImpl ->Controller -> View
 
 [SpringBoard]
 Controller -> View -> Controller -> Service/ServiceImpl -> DAO/DAOImpl ->
 
  *Controller준비
  - 공통적인 URI 경로 설정(컨트롤러를 구분 목적)
  - 각 기능별 URI를 설정(+호출방식 GET/POST)
  - 연결된 뷰페이지 처리(정보전달Model)
  
  * Service 준비
  - 컨트롤러와 DAO를 연결 기능을 설정
  
  * DAO 준비
  -DB연결정보 처리 설정(root-context.xml)
  -Mysql에 데이터베이스/테이블 생성
  -SQL구문(mapper)준비
  
  
  * DB생성
  
  CREATE TABLE `tbl_board` (
  `bno` int NOT NULL AUTO_INCREMENT COMMENT '게시판 글 번호를 설정\nAI(자동으로 숫자를 증가시켜가면서 설정) ',
  `title` varchar(200) NOT NULL COMMENT '글 제목',
  `content` text COMMENT '글 내용',
  `writer` varchar(50) NOT NULL COMMENT '글쓴이',
  `regdate` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '작성일',
  `viewcnt` int DEFAULT '0' COMMENT '조회수 (기본값 0)',
  PRIMARY KEY (`bno`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci
  
  
  * 게시판 만들기 프로젝트
  - 컨트롤러는 /board/*로 매핑한다
  
  - 기능정의
  * 동작에 따른 처리 방식을 결정
     GET 방식 : 조회 / 출력 / 입력 동작 수행시 
     POST 방식 : 처리(등록,수정,삭제...) 동작 수행시


  - 멤버 : 회원가입 -> 게시판 : 글쓰기
  
  
  - 게시판 글쓰기 
  	1) 사용자가 글 정보를 입력하는 페이지 /GET / 주소 : board/regist
  	2) 입력된 글 정보를 처리하는 페이지 /POST / 주소 : board/regist
  	
  - 게시판 목록 보기(조회)
  1) 컨트롤러에 리스트 출력하는 동작 페이지 생성
  
  2) 서비스 -> DB 데이터 가져오기 
  
  3) 연결된 뷰페이지에 출력 (+템플릿)
  
  - 게시판 본문(내용)보기
  - 게시판 내용 수정
  - 게시판 내용 삭제

  Model객체 (택배아저씨)
  :스프링 제공 객체(컨트롤러의 데이터를 뷰페이지로 전달)
  	addAttribute("이름",값);=> 주소줄에 정보 전달 & 정보 유지
  	
  RedirectAttribute 객체(쿠팡아저씨)
  : 스프링 제공 객체(컨트롤러의 데이터를 뷰페이지로 전달)
	단, 페이지를 리다이렉트 이동시 사용가능
  addAttribute("이름",값);
  addFlashAttribute("이름",값);=> 주소줄에 정보 전달x & 정보 유지x (새로고침시 삭제)
  	
  @ModelAttribute
  
  @RequesstParam
  
  RedirectAttirbute 객체
  
  
  
  ------------------------------
  *페이지 블럭
   - 시작 페이지 번호(startPage)
   - 끝 페이지번호(endPage)
   - 전체 데이터의 개수(totalCount)
   - 이전페이지(prev)
   - 다음페이지(next)
   
   ex) 한 페이지에 10개씩 출력, 총 글 122개, 페이지 블럭 크기 : 10개
   -> 총페이지수 12.2개 => 13개
   1) 3page 글 리스트
      startPage =1, endPage = 10, prev = X, next = 0
   2) 10페이지 글 리스트
   3) startPage = 1 , endPage=10, prev=X, next =0
   
  
  
  
  의존 객체 자동주입(Automatic Dependency Injection)
  => 스프링 설정 파일(root-context.xml/servlet-context.xml) 태그를 사용해서
  의존 객체 대상을 명시하지 않고 스프링 컨테이너가 자동으로 객체 주입해주는 것
